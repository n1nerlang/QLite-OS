# Contributing

Thank you for taking the time to improve QLite-OS.

## Before you start

- Keep changes small and focused.
- Follow the existing freestanding style in `boot/`, `c/`, and `rust/`.
- Prefer text-mode demos and simple deterministic behavior.

## Suggested workflow

- Open an issue or describe the change clearly.
- Make the smallest useful patch.
- Validate with the existing checks when possible.

## Pull request expectations

- Explain what changed and why.
- Note any toolchain or environment limitations.
- Keep docs in sync with code where relevant.
